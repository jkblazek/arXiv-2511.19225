## Structure

```
psp/
  market.py      PSP mechanics: valuations θ_i, allocations, PSP cost,
                 ε-best-reply (multi_auction_eps_best_reply) and exact
                 best-response (exact_best_response)
  network.py     Async event-driven simulation engine (priority-queue scheduler)
  ladder.py      Price-ladder verification — Lemma 5.2
  metrics.py     Market snapshots, buyer/seller reporting, convergence summaries
  helpers.py     Seller-side graph utilities: shells, p*, winners
  init.py        Market initialization, adjacency generators, seed management
  plot.py        Visualizations: diagnostics, connectivity, utility surfaces,
                 price-vs-connectivity plots
  io.py          File I/O: traj.dat, prices.dat, run.conf

experiments/
  ladder_check.py    Section 6.2 — runs simulation, writes data/ladder_check/
  connectivity.py    Section 6.3 — runs simulation, writes data/connectivity/
  free_lunch.py      Placeholder — fair vs. QJC tie-splitting experiment

analysis/
  ladder_check.py    Loads data/ladder_check/, prints reports and plots
  connectivity.py    Loads data/connectivity/, produces Figure 8

data/                Generated by experiments/ — not committed to git
  ladder_check/
    run.conf          experiment configuration (JSON)
    traj_000.dat      market state snapshot
    ladder.dat        ladder tuples
  connectivity/
    run.conf          experiment configuration (JSON)
    traj_NNN.dat      one trajectory per trial (rows = connectivity levels)
    prices.dat        ensemble summary: e pavg_j pstd_j vtot ctot utot az
```

## Quickstart

```bash
pip install -r requirements.txt

# Price-ladder verification (Section 6.2)
python -m experiments.ladder_check

# Connectivity sweep (Section 6.3 / Figure 8)
python -m experiments.connectivity
```

## Key design decisions

**Update rule.** The simulation uses `multi_auction_eps_best_reply`, the
ε-best-reply from Lazar & Semret. This is the convergence-guaranteed update
rule. `exact_best_response` is also available for analysis but is not used
in the simulation loop.

**Tie-splitting.** The default tie-breaking rule is QJC (quantity-proportional),
which preserves PSP/VCG incentive-compatibility. A count-split ("fair") variant
is available via `M["tie_policy"] = "fair"` and is the subject of the planned
`experiments/free_lunch.py`.

**Reproducibility.** All random draws are seeded. The connectivity experiment
uses `base_seed = 20405008`; per-level adjacency seeds are derived as
`base_seed + 1009 * level_index`. Per-buyer bid initialization uses independent
streams keyed on `seed + i * 1_000_003`.
